# FinOps Quick Start Guide

> Get started with FinOps in 30 minutes

## The 30-Minute Learning Path

### Step 1: Understand the Basics (10 minutes)
1. Read: `01-basics/01-what-is-finops.md`
2. Understand: Inform → Optimize → Operate
3. Key takeaway: FinOps = Financial Accountability + Engineering

### Step 2: Implement Quick Wins (15 minutes)

**Action Items:**
- Enable AWS Cost Explorer
- Set up basic tagging: Environment, Team, Project, Owner
- Create AWS Budget alerts
- Identify top 10 cost drivers

### Step 3: Find Immediate Savings (5 minutes)

**Low-Hanging Fruit:**
- Delete unattached EBS volumes
- Release unused Elastic IPs
- Remove old snapshots
- Stop non-production instances after hours

**Expected Savings: 10-20% immediately**

## Your First FinOps Dashboard

Essential Metrics:
- Monthly cloud spend and trend
- Top cost drivers by service
- Optimization opportunities
- Waste identification

## 30-Day FinOps Roadmap

**Week 1: Visibility**
- Enable detailed billing
- Implement tagging
- Set up Cost Explorer

**Week 2: Quick Wins**
- Delete unused resources
- Right-size obvious wastes
- Purchase easy RIs

**Week 3: Automation**
- Auto-start/stop instances
- Cost anomaly detection
- Automated tagging

**Week 4: Accountability**
- Team cost allocation
- Showback reports
- FinOps culture kickoff

**Expected: 25-40% cost reduction in 30 days**

## Common Scenarios

### Scenario: AWS bill doubled
- Check Cost Explorer for spike
- Identify new services or regions
- Review recent deployments

### Scenario: Need to reduce costs by 30%
1. Week 1: Delete waste (10-15%)
2. Month 1: Right-sizing (10-15%)
3. Quarter 1: Reserved capacity (10-20%)

## Success Metrics

**Track These KPIs:**
- Cost per transaction
- RI/SP coverage (>70%)
- Waste percentage (<10%)
- Tagged resources (100%)
